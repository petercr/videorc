# Videorc 0.9.17 Beta 1

Videorc 0.9.17 ships the **Plan 024** batch — root causes from Robin's
support bundle + owner preview repros, each adversarially verified before
the fix.

## Scope

- **S1 grant-toast stack** (`052964af`) — a fresh camera/mic grant restarts
  the backend; the ~1s reconnect no longer paints a stack of unkeyed
  "Backend WebSocket is not connected." toasts. Suppressed while
  `wsStatus !== connected` (the badge narrates it); a connected-state blip
  toasts once, keyed. `refreshBackend` gated on `connected`.
- **S2 version-correct support bundle** (`e221b275`) — `app.version` +
  `health.version` now carry the Electron app version (forwarded from
  `runtimeInfo.version`), not the backend crate version (stuck 0.9.0).
  schema 1→2. `commit`-SHA build step deferred to the Release owner.
- **S3 aspect-safe preview present** (`07f8c84b`) — `contain_dest_rect`
  letterboxes the composited frame into the drawable instead of
  stretch-to-fill, killing the ~3% start-stretch; 1:1 when aspects agree.
- **S4 hint-flash fix** (`9091486d`) — `updatePreviewWindowWaitDetail`
  toggles the `.hint` container's display, so a click that raises the
  preview above the order-above helper uncovers only the solid base, never
  the "Waiting for preview" words.
- **S5 camera-format warning** (`7c271796`) — a degenerate single-format
  capture camera (Cam Link 4K@25) shows an actionable Sources-tab warning
  instead of a silent fps fallback.
