# Videorc 0.9.14 Beta 1

Videorc 0.9.14 ships the **full-app QA fix batch** (Plan 022) plus the
console EPIPE crash guard.

## Scope

- **Comments window crash** (Q0, `5dd94a30`) — `useRef` unimported in the
  comments entrypoint; the detached reader error-boundaried on any pushed
  transcript. The gate hole that shipped it is closed: `tsconfig.web.json`
  now typechecks ALL renderer entrypoints (comments/, captions/).
- **Toast policy** (Q3, `c45798c4`) — toaster inset above the footer bar,
  keyed reconciliation toasts, durable Sources alert.
- **System Access truncation** (Q4, `a683d0e2`); **light-theme contrast
  token pass** (Q5, `caeb40cd`); **selects always name their state**
  (Q6, `06046bd5`); **canonical compact labels + 2K tier**
  (Q7, `f69cd283`+`34f308a1`); **comment-highlight static import**
  (Q8, `2ce03900`).
- **Console EPIPE guard** (`057a6cf4`) — a dead stdout/stderr pipe emitted
  async 'error' events with no listener; the "JavaScript error in the main
  process / write EPIPE" dialog can no longer fire from console writes.
- **RC gates documented** (Q1/Q2 agent side, `d3f7f292`) — device-smoke
  sequence + strict provider readiness in the release runbook.
